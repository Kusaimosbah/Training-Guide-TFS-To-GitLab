using System.Net;
using System.Net.Http.Json;
using FluentAssertions;
using Microsoft.AspNetCore.Mvc.Testing;
using CapstoneApi;
using CapstoneApi.Models;

namespace CapstoneApi.IntegrationTests;

public class ProductsControllerTests : IClassFixture<WebApplicationFactory<Program>>
{
    private readonly HttpClient _client;

    public ProductsControllerTests(WebApplicationFactory<Program> factory)
    {
        _client = factory.CreateClient();
    }

    [Fact]
    public async Task GetAllProducts_ReturnsOkWithProducts()
    {
        // Act
        var response = await _client.GetAsync("/api/products");

        // Assert
        response.StatusCode.Should().Be(HttpStatusCode.OK);
        
        var products = await response.Content.ReadFromJsonAsync<List<Product>>();
        products.Should().NotBeNull();
        products.Should().NotBeEmpty();
    }
}

7Understand WebApplicationFactory

‚úÖ What WebApplicationFactory Does:
  • Starts your API in-memory (no real server needed!)
  • Creates an HttpClient to make requests
  • Runs the ENTIRE API stack (controllers, services, middleware)
  • Resets state between tests
üìå IClassFixture: Ensures WebApplicationFactory is created once per test class for performance.

13Add DELETE Test (Complete Workflow)

[Fact]
public async Task DeleteProduct_ValidId_ReturnsNoContent()
{
    // Arrange - First create a product to delete
    var product = new Product
    {
        Name = "To Delete",
        Description = "Will be deleted",
        Price = 1.00m,
        CategoryId = 1
    };
    var createResponse = await _client.PostAsJsonAsync("/api/products", product);
    var createdProduct = await createResponse.Content.ReadFromJsonAsync<Product>();

    // Act - Delete it
    var deleteResponse = await _client.DeleteAsync($"/api/products/{createdProduct!.Id}");

    // Assert
    deleteResponse.StatusCode.Should().Be(HttpStatusCode.NoContent);
    
    // Verify it's gone
    var getResponse = await _client.GetAsync($"/api/products/{createdProduct.Id}");
    getResponse.StatusCode.Should().Be(HttpStatusCode.NotFound);
}
‚úÖ Complete Flow Test: Create ‚Üí Delete ‚Üí Verify deletion!

14Compare Unit vs Integration Tests

Aspect Unit Tests Integration Tests
Scope Single method/class Full API endpoint
Speed Very fast (ms) Slower (100ms+)
Dependencies Mocked/isolated Real (in-memory)
What's Tested Business logic HTTP behavior, routing, serialization
Example ProductService.GetAllProducts() GET /api/products HTTP request

15Understand When to Use Each

üéØ Testing Strategy:
  • Unit Tests: Use for business logic, algorithms, calculations
  • Integration Tests: Use for API contracts, HTTP behavior, end-to-end flows
  • Both Together: Comprehensive coverage!

Typical Ratio: 70% unit tests, 30% integration tests
