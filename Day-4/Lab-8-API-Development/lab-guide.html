Service implements business logic:

public class ProductService : IProductService
{
    private readonly InMemoryDataStore _dataStore;
    
    public ProductService(InMemoryDataStore dataStore)
    {
        _dataStore = dataStore;
    }
    
    public async Task<List<Product>> GetAllProductsAsync()
    {
        // Business logic here
        return await Task.FromResult(_dataStore.Products.ToList());
    }
}
‚úÖ Separation of Concerns: Business logic is isolated from HTTP concerns!

5Explore Controllers - Presentation Layer

cat src/ProductCatalogApi/Controllers/ProductsController.cs

Example controller structure:

[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    private readonly IProductService _productService;
    
    public ProductsController(IProductService productService)
    {
        _productService = productService;  // Injected!
    }
    
    [HttpGet]
    public async Task<ActionResult<List<Product>>> GetAll()
    {
        var products = await _productService.GetAllProductsAsync();
        return Ok(products);
    }
    
    [HttpGet("{id}")]
    public async Task<ActionResult<Product>> GetById(int id)
    {
        var product = await _productService.GetProductByIdAsync(id);
        if (product == null)
            return NotFound();
        return Ok(product);
    }
}

Key Concepts:

  • [ApiController] - Enables automatic model validation, binding
  • [Route] - Defines URL pattern
  • [HttpGet] - Maps to HTTP GET method
  • ActionResult - Flexible return type (Ok, NotFound, BadRequest, etc.)
  • Constructor injection - Service is injected automatically

    Visual structure (implementation):

{
    // ... existing methods (GetAllProductsAsync, GetProductByIdAsync, etc.) ...
    
    // ADD THIS NEW implementation HERE:
    public async Task DeleteProductAsync(int id)
    {
        // ... method code ...
    }

    public async Task DeleteProductAsync(int id)
{
    var product = _dataStore.Products.FirstOrDefault(p => p.Id == id);
    if (product != null)
    {
        _dataStore.Products.Remove(product);
    }
    await Task.CompletedTask;
}

    13Understand Data Flow

    üìä Request Flow:
    1. HTTP Request ‚Üí Arrives at controller endpoint
    2. Controller ‚Üí Validates input, calls service method
    3. Service ‚Üí Executes business logic
    4. Data Store ‚Üí Performs CRUD operation
    5. Service ‚Üí Returns data to controller
    6. Controller ‚Üí Formats HTTP response (Ok, NotFound, etc.)
    7. HTTP Response ‚Üí Returned to client

    14Understand Dependency Injection Benefits

    ‚úÖ Why DI is Powerful:
    • Testability: Easy to mock services for unit tests
    • Flexibility: Swap implementations without changing controllers
    • Maintainability: Single responsibility - controllers handle HTTP, services handle logic
    • Decoupling: Controllers don't create service instances
