üîß Install git-tfs (Windows Only)

‚ö†Ô∏è Important Platform Requirement:

git-tfs requires Windows with Visual Studio or Visual Studio Build Tools installed. It does NOT work reliably on Linux/WSL/Mac.

For this lab: Use PowerShell or Command Prompt in VSCode terminal (not WSL/Bash). Click the terminal dropdown and select "Command Prompt" or "PowerShell".

If you're on Linux/Mac: You can skip this lab and move directly to Lab 2. The core Git and GitLab workflows (Labs 2-10) work perfectly on all platforms and are the main focus of this training.

Installation on Windows (Manual Method - No Admin Required)

# Step 1: Download git-tfs # Go to: https://github.com/git-tfs/git-tfs/releases/download/v0.34.0/GitTfs-0.34.0.zip # Or use PowerShell to download: Invoke-WebRequest -Uri "https://github.com/git-tfs/git-tfs/releases/download/v0.34.0/GitTfs-0.34.0.zip" -OutFile "$env:TEMP\GitTfs-0.34.0.zip" # Step 2: Extract to your user directory (no admin needed) Expand-Archive -Path "$env:TEMP\GitTfs-0.34.0.zip" -DestinationPath "$env:USERPROFILE\git-tfs" -Force # Step 3: Add to PATH for current session $env:PATH = "$env:USERPROFILE\git-tfs;$env:PATH" # Step 4: Verify installation git tfs --version # Should output: git-tfs version 0.34.0.0 # Step 5: (Optional) Add to PATH permanently # Open: Windows Settings ‚Üí System ‚Üí About ‚Üí Advanced System Settings ‚Üí Environment Variables # Edit "Path" under "User variables" and add: C:\Users\YourUsername\git-tfs
üí° Alternative Methods (Require Admin Rights):
  • Chocolatey: choco install gittfs
  • Scoop: scoop install git-tfs

If you get permission errors with package managers, use the manual method above.

Step 2: Understanding Where the Conversion Happens

üîç Critical Concept: Where TFVC ‚Üí Git Conversion Occurs

üìç IN THIS LAB:
  • ‚ùå No conversion happens - we skip it!
  • ‚úÖ setup-repo.ps1 directly creates a Git repository
  • ‚úÖ We pretend this Git repo came from TFVC
  • ‚úÖ We practice the GitLab migration part only
üìç IN REAL TFS MIGRATION:

‚öôÔ∏è Conversion happens INSIDE git-tfs tool:
Step 1: git-tfs connects to TFS server
Step 2: Downloads TFVC changeset 1001 ‚Üí git commit with message "TFS Changeset 1001"
Step 3: Downloads TFVC changeset 1002 ‚Üí git commit with message "TFS Changeset 1002"
Step 4: Downloads TFVC changeset 1003 ‚Üí git commit with message "TFS Changeset 1003"
Step 5: Downloads TFVC changeset 1004 ‚Üí git commit with message "TFS Changeset 1004"
Result: A Git repository with 4 commits (same as what we have now!)
üéØ What We're Teaching You:
  • NOT: How to convert TFVC to Git (git-tfs does this automatically)
  • YES: What to do AFTER git-tfs finishes (migrate to GitLab)
  • YES: How to verify the migration worked
  • YES: How to push history to GitLab
üí° Think of it this way:
setup-repo.ps1 = "Fast-forward to the point where git-tfs is already done"

Step 2: Create GitLab Project

Create a new project on your GitLab server:

  1. Open GitLab: https://gitlab.alleen-ly.online
  2. Click "New project" ‚Üí "Create blank project"
  3. Project name: legacy-app-migrated
  4. Visibility: Private
  5. UNCHECK "Initialize repository with a README"
  6. Click "Create project"
  7. Copy the Git URL: https://gitlab.alleen-ly.online/your-username/legacy-app-migrated.git

Step 3: Push to GitLab

Push the migrated repository to GitLab:

# Add GitLab as remote (replace with your GitLab URL) git remote add origin https://gitlab.alleen-ly.online/your-username/legacy-app-migrated.git # Rename master to main (GitLab default) git branch -M main # Push all commits to GitLab git push -u origin main # Enter your GitLab credentials when prompted
‚úÖ Migration Complete!
Your "TFS repository" is now on GitLab with full history preserved. Go to GitLab and verify:
  • All 4 commits are visible
  • Commit messages show "TFS Changeset" numbers
  • All files are present

Step 4: Verify Migration

Check GitLab to ensure everything migrated correctly:

  1. Go to your project in GitLab
  2. Click "Repository" ‚Üí "Commits"
  3. Verify all 4 commits are there with correct messages
  4. Click on each commit to see file changes
  5. Check "Repository" ‚Üí "Files" to see all project files

üõ†Ô∏è Troubleshooting Common Issues

Issue 1: PowerShell Execution Policy Error

Error: Cannot run setup-repo.ps1 - execution policy restricted
# Check current execution policy Get-ExecutionPolicy # Allow script for current session only Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass # Then run the script .\setup-repo.ps1

Issue 2: Git Not Initialized

Error: "fatal: not a git repository"
# Make sure you ran the setup script .\setup-repo.ps1 # Verify .git folder exists Test-Path .git # Check Git status git status

Issue 3: GitLab Push Authentication Failed

Error: Authentication failed when pushing to GitLab
# Verify GitLab URL is correct git remote -v # Update remote URL if needed git remote set-url origin https://gitlab.alleen-ly.online/your-username/legacy-app-migrated.git # Use Personal Access Token instead of password # Generate PAT in GitLab: Settings ‚Üí Access Tokens # Username: your-username # Password: paste-your-PAT-token git push -u origin main

Issue 4: Application Won't Run

Error: "dotnet run" fails
# Check .NET SDK is installed dotnet --version # If not installed, download .NET 8 SDK from: # https://dotnet.microsoft.com/download # Restore dependencies dotnet restore # Build project dotnet build # Run application dotnet run

Issue 5: Can't See Commit History in GitLab

Error: GitLab shows only 1 commit instead of 4
# Verify local commits exist git log --oneline --all # Make sure you pushed all commits git push origin main --force # Check GitLab project ‚Üí Repository ‚Üí Commits # Should show 4 commits with "TFS Changeset" messages

Real TFS Migration Issues (Reference)

üí° When migrating from actual TFS servers:
  • Connection timeout: Use git tfs clone --username --password
  • Large repos: Use --changeset=XXXXX to start from recent history
  • Missing branches: Use git tfs branch --init to fetch specific branches
  • Authentication: Azure DevOps requires Personal Access Token (PAT)

üìä Migration Best Practices

Before Migration

  • ‚úÖ Create backup of TFS repository
  • ‚úÖ Document all active branches
  • ‚úÖ List all team members (for author mapping)
  • ‚úÖ Check repository size and history depth
  • ‚úÖ Clean up unused branches and shelvesets in TFS
  • ‚úÖ Test migration on a small project first

During Migration

  • ‚úÖ Use author mapping file for proper Git attribution
  • ‚úÖ Migrate in off-hours to avoid conflicts
  • ‚úÖ Keep TFS repository in read-only mode
  • ‚úÖ Monitor disk space during clone
  • ‚úÖ Document any issues encountered

After Migration

  • ‚úÖ Verify all commits migrated
  • ‚úÖ Check all branches exist in GitLab
  • ‚úÖ Test build and CI/CD pipelines
  • ‚úÖ Train team on Git workflows
  • ‚úÖ Update documentation and wiki links
  • ‚úÖ Keep TFS in read-only for reference period
